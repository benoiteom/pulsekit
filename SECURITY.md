# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | Yes                |
| < 1.0   | No                 |

## Reporting a Vulnerability

If you discover a security vulnerability in PulseKit, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please [open a GitHub Security Advisory](https://github.com/benoiteom/pulsekit/security/advisories/new) or contact **[@benoiteom](https://github.com/benoiteom)** directly on GitHub with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

You should receive an acknowledgment within 48 hours. We will work with you to understand the issue and coordinate a fix before any public disclosure.

## Scope

This policy applies to all packages in the PulseKit monorepo:

- `@pulsekit/core`
- `@pulsekit/next`
- `@pulsekit/react`
- `create-pulsekit`
